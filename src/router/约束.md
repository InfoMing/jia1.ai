# src/router 路由编码约束

## 1. 配置规范

- 使用 `new Router()` 语法（Vue Router 3.x）
- 路径使用 kebab-case
- 所有页面组件使用懒加载：`component: () => import('@/admin/views/...')` 或 `component: () => import('@/business/views/...')`
- 布局路由统一使用 `Layout` 组件作为父级
- 路由定义按域拆分：后台路由在 `router/admin.js`，前台路由在 `router/business.js`，`router/index.js` 只做合并

## 2. 路由结构

```js
{
  path: '/system',
  component: Layout,
  alwaysShow: true,
  meta: { title: '系统', icon: 'setting' },
  children: [
    {
      path: 'schedule',
      meta: { title: '定时任务', icon: 'schedule' },
      component: () => import('@/admin/views/system/schedule/index.vue')  // 或 @/business/views/...
    }
  ]
}
```

## 3. meta 字段规范

- `title`：中文页面标题
- `icon`：对应 SVG 图标名称
- `affix`：是否固定标签页（Boolean）
- `noCache`：是否不缓存（Boolean）

## 4. 特殊路由

- 404、401 等错误页设 `hidden: true`
- 登录页不包在 Layout 内，设 `hidden: true`
- 使用 `resetRouter()` 函数重置路由实例

## 注释要求

- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
