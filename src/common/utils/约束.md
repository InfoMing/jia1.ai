# web/src/utils 工具函数编码约束

## 1. 用户信息缓存规范

- 用户信息统一存储在 `sessionStorage` 或 `localStorage` 中，key 固定为 `LOCAL_CACHE_USER_INFO_KEY`
- 工具函数统一放在 `web/src/utils/user.js` 中

### 1.1 存储规则

| 场景 | 存储位置 | key |
|---|---|---|
| 未勾选「记住密码」登录 | `sessionStorage` | `LOCAL_CACHE_USER_INFO_KEY` |
| 勾选「记住密码」登录 | `sessionStorage` + `localStorage` | `LOCAL_CACHE_USER_INFO_KEY` |
| 注册后自动登录 | `sessionStorage` | `LOCAL_CACHE_USER_INFO_KEY` |

### 1.2 Token 获取

- `getToken()`：优先从 `sessionStorage` 读取 `LOCAL_CACHE_USER_INFO_KEY` 中的 token；未找到则从 `localStorage` 读取
- `getUserInfo()`：格式统一为 `{ token, user: { loginCode, ... }, resources, ... }`

### 1.3 退出登录

```js
function logout() {
  sessionStorage.clear()
  window.localStorage.clear()
  location.reload()
}
```

## 2. API 工具函数规范

- 登录/注册/找回密码等用户认证 API 统一放在 `web/src/api/user.js` 中
- 方法命名使用 camelCase，动词开头
- 使用 `request()` 封装 Axios 实例
- `userapi` 路径前缀：`const userapi_baseUrl = "/userapi/user"`
- POST 请求使用 `data` 传参

```js
// 登录
login: (data) => request({ url: userapi_baseUrl + '/login', method: 'post', data })

// 注册
register: (data) => request({ url: userapi_baseUrl + '/register', method: 'post', data })

// 找回密码
forgotPassword: (data) => request({ url: userapi_baseUrl + '/forgotPassword', method: 'post', data })
```

## 3. 导航守卫规范

- 路由导航守卫在 `permission.js` 中统一管理
- Token 检查通过 `getToken()` 判断登录状态
- 白名单路由：公开页面（首页、登录、注册、找回密码、市场行情）无需登录
- 非白名单且以 `/business/` 开头的路由 → 自动重定向到 `/business/login?redirect=<path>`

## 注释要求

- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
