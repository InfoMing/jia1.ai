# src/store Vuex 状态管理编码约束

## 1. 模块规范

- 后台模块定义在 `admin/store/` 下，前台模块定义在 `business/store/` 下
- 必须设置 `namespaced: true`
- 导出结构：`{ namespaced, state, mutations, actions }`
- 由顶层 `store/index.js` 显式 import 注册

## 2. 命名规范

- mutations：`SCREAMING_SNAKE_CASE`，如 `SET_USER`、`TOGGLE_SIDEBAR`
- actions：camelCase，如 `toggleSideBar`、`generateRoutes`
- state：camelCase 属性

## 3. 调用规范

- 组件中：`this.$store.dispatch('moduleName/actionName')`
- 组件映射：`...mapState` 或 `...mapGetters`
- 工具函数中可直接 `store.commit('module/MUTATION', data)`

## 注释要求

- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
