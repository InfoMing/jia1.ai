# admin/api/ 后台 API 请求层编码约束

## 1. 文件规范

- 每个功能域一个 JS 文件，如 `user.js`、`code.js`
- 默认导出对象，包含所有 API 方法

## 2. 方法规范

- 方法名 camelCase，HTTP 动词对应操作
- URL 前缀常量：`const adminapi_baseUrl = "/adminapi/user"`
- POST 用 `data` 传参，GET 用 `params` 传参

## 3. 域隔离

- 本目录存放后台管理 API，不得被 `business/` 域引用
- 前台业务 API 放在 `business/api/`

## 注释要求

- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
