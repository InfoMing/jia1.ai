# admin/constants/ 后台常量编码约束

## 1. 文件规范

- `settings.js` — 后台布局配置常量（showSettings、tagsView、fixedHeader 等）
- 常量命名使用 `camelCase`

## 2. 域隔离

- 本目录存放后台专有常量，不得被 `business/` 域引用
- 前台常量放在 `business/constants/`

## 注释要求

- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
