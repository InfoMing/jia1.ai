# admin/store/ 后台 Vuex 状态管理编码约束

## 1. 模块规范

- 本目录存放后台专有 Vuex 模块
- 当前模块：`app`、`base`、`errorLog`、`permission`、`settings`、`tagsView`、`user`
- 每个模块文件名为 `camelCase.js`，必须设置 `namespaced: true`
- 由顶层 `store/index.js` 显式导入

## 2. 命名规范

- mutations：`SCREAMING_SNAKE_CASE`
- actions：camelCase
- state：camelCase 属性

## 3. 域隔离

- 后台 store 模块不得被 `business/` 域直接引用
- 前台若需新增 Vuex 模块，放在 `business/store/`

## 注释要求

- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
