# admin/ 后台管理域编码约束

> 继承自：[web/约束.md](../../约束.md)

## 1. 目录职责

- `layout/` — 后台布局（侧边栏 + 顶部导航）
- `views/` — 后台页面（系统管理、资源管理、平台管理等）
- `api/` — 后台 API 调用
- `store/` — 后台 Vuex 状态
- `components/` — 后台通用组件（BaseTable、BaseTree 等）
- `styles/` — 后台样式
- `constants/` — 后台常量
- `assets/` — 后台静态资源

## 2. 域隔离原则

- `admin/` 下的代码不得引用 `business/` 目录下的模块
- `admin/` 只能引用 `common/`（工具函数、过滤器）和自身内部模块
- 引用路径使用 `@/admin/xxx` 别名

## 3. 路由规范

- 后台路由统一以 `/admin` 为前缀
- 路由定义在 `router/admin.js` 中

## 4. API 调用规范

### 4.1 分页查询
- 分页查询统一使用 `POST` 方法 + `data` 参数（JSON body），格式为 `PageRequestBodyDto`
- 禁止使用 `GET` 方法 + `params` 方式调用分页接口
- 示例：
  ```javascript
  // ✅ 正确
  getListByObj: (obj) => request({ url: '/path/list', method: 'post', data: obj })

  // ❌ 错误
  getListByObj: (obj) => request({ url: '/path/list', method: 'get', params: obj })
  ```

### 4.2 更新操作
- 更新操作统一使用 `PUT` 方法，以 `code` 作为标识路径参数（如 `PUT /{code}`）
- 不使用 `id` 作为标识路径参数

### 4.3 删除操作
- 删除操作统一使用 `DELETE` 方法，以 `code` 作为标识路径参数（如 `DELETE /{code}`）
- 不使用 `id` 作为标识路径参数

## 注释要求

- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
