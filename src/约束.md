# web/src/ 前端源码编码约束

## 1. 导入规范

- 使用 `@/` 别名引用 `src/` 下模块
- 单引号，无分号
- 缩进 2 个空格
- 导入顺序：Vue/第三方库 → Element UI → 全局样式 → 应用模块 → 工具函数

## 2. 入口文件规范

- `main.js` 注册全局插件、过滤器、指令、原型属性
- 原型属性加 `$` 前缀：`Vue.prototype.$cookie`、`Vue.prototype.$code`
- 全局过滤器通过迭代注册

## 3. 权限控制

- 路由导航守卫在 `permission.js` 中统一管理
- Token 检查通过 `getToken()` 判断登录状态
- 动态路由从后端获取菜单后生成
- 白名单路由：`/login`、`/auth-redirect` 等

## 4. 目录结构

```
src/
├── admin/           # 后台所有专有代码（layout、views、components、styles、api、store、assets）
├── business/        # 前台所有专有代码（layout、views、components、styles、api、store、assets）
├── common/          # 双方共享的纯工具（utils、filters）
├── router/          # 路由配置（index.js 合并 admin.js + business.js）
├── store/           # Vuex 组装（显式导入 admin/store + business/store）
├── permission.js    # 路由守卫（内部分区：前台白名单 / 后台权限检查）
├── App.vue
└── main.js
```

## 5. 域隔离原则

- `admin/` 下代码不得被 `business/` 引用
- `business/` 下代码不得引用 `admin/`
- `common/` 下的模块可以被双方引用，但不得引用 `admin/` 或 `business/`
- 引用路径使用 `@/{域}/` 别名

## 6. 环境配置

- `.env.development`、`.env.production`、`.env.staging` 分别配置
- Axios baseURL 通过 `process.env.VUE_APP_BASE_API` 读取

## 注释要求

- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
