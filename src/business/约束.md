# business/ 前台域编码约束

## 1. 目录职责

- `layout/` — 前台公共布局（导航栏、页脚、登录）
- `views/` — 前台页面及页面专用组件
- `api/` — 前台 API 调用
- `mock/` — 开发阶段后端响应模拟，禁止由页面直接引用
- `styles/` — 前台公共样式
- `components/` — 多页面复用的前台公共组件
- `constants/` — 前台常量
- `store/` — 前台 Vuex 模块（预留）
- `assets/` — 前台静态资源（预留）

## 2. 域隔离原则

- `business/` 下的代码不得引用 `admin/` 目录下的模块
- `business/` 只能引用 `common/`（工具函数、过滤器）和自身内部模块
- 引用路径使用 `@/business/xxx` 别名

## 3. 路由规范

- 前台路由使用无 `.html` 的 kebab-case 规范路径
- 路由定义在 `router/business.js` 中
- 页面组件必须懒加载自 `business/views/`
- 旧 `.html` 地址只能作为重定向或 iframe 内部 source

## 4. API 调用规范

- 页面只能调用 `business/api`，不得直接引用 `business/mock`
- 统一响应结构为 `{ code, message, data }`，成功码为 `2000`

- 分页查询统一使用 `POST` 方法 + `data` 参数（JSON body），格式为 `PageRequestBodyDto`
- 禁止使用 `GET` 方法 + `params` 方式调用分页接口
- 示例：
  ```javascript
  // ✅ 正确
  list: (params) => request({ url: '/path/list', method: 'post', data: params })

  // ❌ 错误
  list: (params) => request({ url: '/path/list', method: 'get', params })
  ```

## 注释要求

- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
