# views/business/login/ 登录注册找回密码编码约束

## 1. 样式共享

- 登录、注册、找回密码三个页面共享同一套卡片样式
- 公共样式抽取到 `_login-common.scss` 中，各页面通过 `@import` 引入
- 页面容器统一使用类名 `.login-page`（flex 居中，`background: #F5F6FA`）
- 卡片统一使用类名 `.login-card`（白色背景，`border-radius: 12px`，`box-shadow`）

### 1.1 公共样式文件内容

- `.login-page` — 页面容器
- `.login-card` — 卡片容器
- `.login-form-panel` — 表单面板
- `.login-form` — 表单（`max-width: 220px`）
- `.input-wrap` — 输入框包裹容器（flex 行，边框 `1px solid #E4E6EB`，focus `#C62F2E`）
- `.submit-btn` — 主按钮（全宽 38px，`background: #C62F2E`，hover `#A82828`）
- `.code-wrap` / `.code-btn` — 验证码按钮
- `.form-bottom` / `.form-link` — 底部链接
- `.page-title` / `.page-subtitle` — 页面标题

### 1.2 找回密码页面布局

找回密码页面**无左侧品牌面板**，仅保留右侧表单面板，居中显示。
标题使用「合适价投」品牌文字 +「找回密码」副标题。

## 2. 找回密码 3 步流程

整个找回密码流程在同一页面通过 `step` 变量驱动，不切换路由：

```
step: 'identity'  →  step: 'verify'  →  step: 'reset'
```

### Step 1: identity

- 输入账号/手机号/邮箱，点击「下一步」
- 至少需要填写一项

### Step 2: verify

- 选择接收方式：手机号 / 邮箱（radio 或 tab 切换）
- 填写完整收件地址（手机号或邮箱）
- 填写部分位验证：
  - 若选择手机号：填写后 4 位，与完整手机号匹配方可获取验证码
  - 若选择邮箱：填写 @ 前字符，与完整邮箱匹配方可获取验证码
- 验证码固定 `8888`，前端校验
- 验证码按钮带 60 秒倒计时

### Step 3: reset

- 输入验证码 + 新密码 + 确认密码
- 点击「重置密码」调用后端 API
- 成功后跳转登录页

## 3. 登录表单

- 密码登录（`loginMode: 1`）：支持输入账号或手机号
- 验证码登录（`loginMode: 2`）：输入手机号 + 验证码
- 密码登录表单添加「记住密码」checkbox
- 开发阶段默认填入 `account: 'admin'`、`password: '123456'`
- 「找回密码」链接指向 `/business/forgot-password` 路由

## 4. 注册表单

- account（账号）为选填，placeholder 显示「账号（选填，数字+字母组合）」
- phone（手机号）为必填
- 注册成功 → 自动调用登录 API → 跳转 `/business/market`

## 5. 错误处理

- API 调用失败使用 `this.$message.error('中文错误消息')` 提示
- 表单校验规则通过 `el-form` 的 `:rules` 属性声明
- 登录/注册按钮加载状态用 `loading` 变量控制

## 6. 记忆密码实现

- 勾选「记住密码」：登录成功后 `account` + `password` 存入 `localStorage`
- 页面加载时检查 `localStorage`，有则自动填充到表单
- 未勾选：不存 `localStorage`，关闭页面即失效
