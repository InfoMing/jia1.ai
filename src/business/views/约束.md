# views/business/ 业务视图编码约束

## 1. 目录隔离

- 业务视图统一放在 `views/business/` 下，不得与 `views/resource/`、`views/system/` 等管理端目录混用
- 业务 API 调用统一放在 `api/business/` 下，不得与 `api/baseapi/` 混用
- 业务组件放在 `views/business/` 内的子目录中，不得引用 `components/common/` 以外的管理端组件

## 2. 路由隔离

- 业务路由使用独立的路由配置文件 `router/business.js`，不得写入 `router/index.js`
- 业务路由路径统一以 `/business` 为前缀（首页 `/` 除外）
- 业务路由懒加载路径统一使用 `@/business/views/` 前缀

## 3. 样式隔离

- 业务页面使用独立样式变量，不得引用管理端的 `styles/variables.scss` 中的业务无关变量
- 业务组件样式使用 `scoped` 隔离，避免污染管理端页面

## 4. 注释要求

- 业务逻辑使用中文注释
- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 5. 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
