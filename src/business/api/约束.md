# api/business/ 业务 API 编码约束

## 1. 目录隔离

- 业务 API 调用统一放在 `api/business/` 下，不得与 `api/baseapi/` 混用
- 每个业务模块一个 JS 文件，如 `monitor.js`、`security.js`

## 2. 路径规范

- 业务用户端接口使用 `userapi` 路径前缀
- 业务管理端接口使用 `adminapi` 路径前缀
- API 方法命名使用 camelCase，动词开头

## 3. 注释要求

- 每个 API 方法使用中文注释说明功能
- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明

## 4. 错误处理

- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用
