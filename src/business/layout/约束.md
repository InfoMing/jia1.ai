# business/layout 前台公共布局约束

## 1. 公共职责

- 导航、页脚、登录注册等跨页面模块放在 `layout/components/`。
- 页面专用内容不得放入 layout，应放在对应 `views/<page>/components/`。
- 公共布局通过 `SITE_SHELL_KEY` 提供登录状态和登录入口，页面不得自行挂载第二套登录弹层。

## 2. 导航

- 导航项由 `business/config/home-content.js` 集中维护。
- 所有站内入口必须使用命名路由，不得拼接 URL 或使用 `.html` 地址。
- 1099px 以下使用上下两行布局；第二行允许横向滚动并自动展示当前路由。

## 3. 数据与错误

- 布局中的登录、退出和点赞统一调用 `business/api`。
- 页面与布局不得直接引用 `business/mock`，也不得直接维护后端业务状态。
- API 错误使用 Element Plus Message 显示中文提示。

## 4. 注释

- 登录跳转、导航滚动等非直观逻辑必须添加中文注释。
