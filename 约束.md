# web/ 前端模块编码约束

## 1. 技术栈

- **框架**：Vue 2.6 + Vue Router 3.0 + Vuex 3.1
- **UI 库**：Element-UI 2.15
- **HTTP**：Axios 0.18
- **样式**：SCSS
- **构建**：Vue CLI 4.4

## 2. 命名规范

### 文件命名
- 视图文件：`kebab-case` 目录 + `index.vue`
- 组件目录：`PascalCase` + `index.vue`
- JS 模块：`camelCase.js`
- SCSS 文件：`kebab-case.scss`
- Vuex 模块：`camelCase.js`
- API 模块：`camelCase.js`

### 代码命名
- 组件 `name` 属性：PascalCase，与目录名一致
- API 方法：camelCase，动词开头
- Vuex mutations：`SCREAMING_SNAKE_CASE`
- Vuex actions：camelCase
- Vuex 模块：必须设置 `namespaced: true`

## 3. 目录结构

- `src/api/` — API 接口模块
- `src/components/common/` — 可复用公共组件
- `src/views/{domain}/` — 页面视图，按功能域分组
- `src/store/modules/` — Vuex 状态模块
- `src/router/` — 路由配置
- `src/utils/` — 工具函数
- `src/styles/` — 全局样式
- `src/layout/` — 布局组件
- `src/filters/` — Vue 过滤器
- `src/assets/` — 静态资源

## 4. 组件规范

- Props 使用对象形式，必填类型和默认值
- 对象和数组默认值使用工厂函数：`default: () => ({})`、`default: () => []`
- 回调使用 `Function` 类型 prop
- 模板中使用 `PascalCase` 组件名
- 使用 `scoped` 样式隔离
- 事件使用原生 DOM 事件或 Element UI 事件，优先使用 prop 回调

## 5. 路由规范

- 使用 `new Router()` 语法（Vue 2）
- 路径使用 kebab-case
- 所有页面路由使用懒加载：`component: () => import('@/views/...')`
- 布局统一用 `Layout` 组件包裹子路由
- meta 字段包含 `title`（中文标题）、`icon`、`affix`、`noCache`

## 6. 样式规范

- 使用 SCSS，`<style lang="scss" scoped>`
- 全局样式变量在 `styles/variables.scss` 中定义
- 变量命名 `$kebab-case`
- 使用 `:export` 实现 JS 与 SCSS 共享变量
- 使用 Element UI 主题变量覆盖 `$--color-*`

## 7. 错误处理

- Axios 拦截器统一处理错误码（`code !== 2000` 时弹 Message）
- 网络错误自动翻译为中文提示
- API 方法内校验参数，不合法时 `throw new Error`
- 报错提示、日志信息、异常消息等使用中文编写，尽量减少英文使用

## 8. 注释

- 业务逻辑使用中文注释
- 组件 JSDoc 使用中文
- 代码应添加适量注释，关键逻辑、复杂算法、接口定义、配置项等必须有注释说明
